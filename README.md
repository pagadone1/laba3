# Car Service — автосервис

Веб-сервис для учёта заказов на ремонт и обслуживание автомобилей. Все операции сохраняются в PostgreSQL.

## Тема

Сервис для автосервиса: учёт клиентов, автомобилей, механиков, запчастей и заказов на обслуживание.

## Основные сущности

| Сущность | Описание | Ограничения |
|----------|----------|-------------|
| **Customer** | Клиент | имя, email (уникальный), телефон |
| **Vehicle** | Автомобиль | номер (уникальный), марка, год, владелец (→ Customer) |
| **Mechanic** | Механик | имя, специализация |
| **Part** | Запчасть | название, цена, количество на складе |
| **ServiceOrder** | Заказ на обслуживание | автомобиль, клиент, механик, запчасти, стоимость работ, статус (OPEN/CLOSED) |
| **OrderPart** | Связь заказ↔запчасть | заказ, запчасть, количество (уникальная пара order_id + part_id) |

## Операции сервиса

### CRUD (все действия сохраняются в БД)
- **Клиенты** `/api/customers` — создать, получить, обновить, удалить
- **Автомобили** `/api/vehicles` — создать, получить, обновить, удалить
- **Механики** `/api/mechanics` — создать, получить, обновить, удалить
- **Запчасти** `/api/parts` — создать, получить, обновить, удалить
- **Заказы** `/api/service-orders` — создать, получить, обновить, удалить

### Бизнес-операции (5 шт.)
1. **Закрытие заказа** — `PUT /api/service-orders/{id}/close` — только если все работы выполнены
2. **Расчёт суммы заказа** — `GET /api/service-orders/{id}/total` — запчасти + работа
3. **Добавление запчастей к заказу** — `POST /api/service-orders/{id}/parts` — body: `[1, 3]` (ID запчастей); списание со склада в одной транзакции
4. **История заказов клиента** — `GET /api/service-orders/by-customer/{customerId}`
5. **Открытые заказы механика** — `GET /api/service-orders/by-mechanic/{mechanicId}/open`

### Дополнительно
- **Автомобили клиента** — `GET /api/vehicles/by-owner/{customerId}`

## Запуск

### Требования
- Java 17+
- PostgreSQL

### 1. Создать базу
```sql
CREATE DATABASE carservice;
```

### 2. Конфигурация (чувствительные данные в .env)
Скопируйте `.env.example` в `.env` и задайте параметры:
```
DB_URL=jdbc:postgresql://localhost:5432/carservice
DB_USERNAME=postgres
DB_PASSWORD=ваш_пароль
```

**Важно:** не добавляйте `.env` в git — файл с паролями не должен попадать в репозиторий.

Экспортируйте переменные перед запуском (PowerShell):
```powershell
$env:DB_URL="jdbc:postgresql://localhost:5432/carservice"
$env:DB_USERNAME="postgres"
$env:DB_PASSWORD="ваш_пароль"
```

### 3. Запуск приложения
```bash
cd demo
mvn spring-boot:run
```

Сервис доступен на http://localhost:8080

### 4. Тестовые данные
При первом запуске `data.sql` загружает тестовых клиентов, механиков, запчасти, автомобили и один заказ. Все операции из Postman сохраняются в БД и переживают перезапуск (ddl-auto=update).
